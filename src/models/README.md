# Models — CS229 MD Prediction

This directory contains all model training, analysis, and utility scripts for predicting multi-state vs. single-state MD trajectory behavior from ESM-IF embeddings.

---

## Where to Run

All scripts are designed to run on the **GCP VM**, which has:
- GPU access (CUDA)
- Local chunk cache at `/home/jupyter/raw_chunk_cache/`
- GCS mount at `/home/jupyter/gcs_mount/`
- All dependencies pre-installed in the conda environment

You can run smaller experiments locally if you have a GPU and the data cached, but GCS-backed data loading will be slow without the VM's network proximity.

### Data

Training requires chunk-level metadata CSVs and the corresponding `.npy` embedding files.

**Metadata CSVs** (generated by `remap_chunks_split.py`):
```
/tmp/emb_bootstrap_train_chunks_newsplit.csv
/tmp/emb_bootstrap_test_chunks_newsplit.csv
```

**Embedding cache** (one file per chunk):
```
/home/jupyter/raw_chunk_cache/train/chunk_<traj_id>_<chunk_id>.npy
/home/jupyter/raw_chunk_cache/test/chunk_<traj_id>_<chunk_id>.npy
```
Each file is a `(T, R, 512)` float16 array — T frames, R residues, 512 ESM-IF dims.

**GCS fallback** (slower):
```
gs://cs229-md/data/raw_esmif_chunks/
```

### Data Split Setup

Before training, generate the chunk-level train/test CSVs from the protein-level split:


## Models

### 1. TICA Eigenvalue Classifier

Fits TICA on each chunk's Cα coordinates, extracts top-k eigenvalues, and sweeps a threshold classifier on trajectory-level mean eigenvalue scores. Does not require ESM-IF embeddings — operates directly on raw Cα coordinates from GCS.

**Quick start (using precomputed eigenvalues):**
```bash
./TICA_threshold/run_tica.sh
```

**Manual:**
```bash
python TICA_threshold/tica_threshold_classifier.py \
  --load-ev notebooks/bootstrap_tica_eigenvalues.csv \
  --n-ev 2 \
  --plot TICA_threshold/tica_plot.png \
  --latex
```

**Recompute eigenvalues from scratch:**
```bash
python TICA_threshold/tica_threshold_classifier.py \
  --lag 10 \
  --n-ev 2 \
  --save-ev eigenvalues.csv \
  --plot TICA_threshold/tica_plot.png \
  --latex
```

**Key flags:**
| Flag | Default | Description |
|---|---|---|
| `--lag` | `10` | TICA lag time in frames |
| `--n-ev` | `2` | Max eigenvalues to compute per chunk; also selects the reported row |
| `--n-ev-list` | `1 2 3 5 10` | Eigenvalue counts to sweep in the metrics table |
| `--save-ev` | — | Save computed eigenvalues to CSV (for caching) |
| `--load-ev` | — | Load precomputed eigenvalues instead of recomputing |
| `--plot` | — | Path to save eigenvalue distribution + threshold plot (PNG) |
| `--latex` | false | Also print LaTeX table source |

**Data sources (hardcoded paths):**
```
notebooks/bootstrap_metadata.csv              # chunk metadata
notebooks/bootstrap_tica_eigenvalues.csv      # precomputed eigenvalues (use with --load-ev)
data/processed/protein_level_train.csv        # train split
data/processed/protein_level_test.csv         # test split
gs://cs229-central/data/processed/v3/         # raw Cα coordinate arrays (GCS)
```

**Outputs:** Console table sweeping over `--n-ev-list`, optional LaTeX table, optional plot with per-eigenvalue histograms and threshold sweep.

### 5. ResAttn Transformer

Two-level attention model: residue-level (α) and temporal (β). Supports learned positional embeddings or ALiBi.

**Quick start:**
```bash
./Transformer/run_transformer.sh my_run_name
```

**Manual:**
```bash
python Transformer/train_resattn_transformer.py \
  --split newsplit \
  --d-inner 512 \
  --d-model 128 \
  --n-heads 4 \
  --n-layers 2 \
  --dim-ff 256 \
  --dropout 0.3 \
  --peak-lr 1e-4 \
  --epochs 80 \
  --patience 6
```

**Key flags:**
| Flag | Description |
|---|---|
| `--alibi` | Use ALiBi relative position bias (causal architecture) |
| `--causal` | Causal attention masking |
| `--d-inner` | Residue projection input dim (default 512) |
| `--d-model` | Transformer hidden dim (128 bidirectional, 64 ALiBi) |
| `--layer-drop` | Layer drop probability |

The `run_transformer.sh` script also runs `analyze_transformer.py` and `analyze_residue_mapping.py` automatically after training.

**Outputs:** `runs_transformer/<run_name>/` with same structure as Mamba, plus `analysis/`.

### 6. Mamba SSM

State-space model operating over frames, with configurable residue pooling.

**Quick start:**
```bash
./Mamba/run_mamba.sh my_run_name
```

**Manual:**
```bash
python Mamba/train_mamba.py \
  --split newsplit \
  --pooling attention \
  --d-model 128 \
  --n-layers 2 \
  --d-state 16 \
  --epochs 80 \
  --peak-lr 1e-4 \
  --patience 6 \
  --label-smooth 0.1
```

**Key flags:**
| Flag | Default | Description |
|---|---|---|
| `--pooling` | `attention` | Residue pooling: `mean`, `meanvar`, `attention`, `conv`, `resattn` |
| `--split` | `newsplit` | Data split preset |
| `--d-model` | 128 | Hidden dimension |
| `--n-layers` | 2 | Number of Mamba layers |
| `--d-state` | 16 | SSM state dimension |
| `--max-frames` | 32 | Truncate chunks to N frames |
| `--max-res` | 300 | Truncate to N residues |

**Outputs:** `runs_mamba/<run_name>/` with `checkpoint_best.pt`, `history.csv`, `results.json`, `config.json`.

## Common Arguments (All Training Scripts)

```
--train-csv      Path to train metadata CSV
--test-csv       Path to test metadata CSV
--local-cache    Directory with cached .npy chunks
--output-dir     Where to write run directories
--split          "newsplit" (protein-level) or "original"
--epochs         Max training epochs
--peak-lr        Peak learning rate
--warmup         LR warmup steps
--patience       Early stopping patience
--micro-bs       Micro-batch size
--accum-steps    Gradient accumulation steps
--label-smooth   Label smoothing (default 0.1)
--max-frames     Truncate chunks to N frames
--max-res        Truncate to N residues
```

Every training run produces:
```

├── checkpoint_best.pt     # Best val-AUC checkpoint
├── checkpoint_latest.pt   # Last epoch
├── config.json            # Full hyperparameter record
├── history.csv            # Epoch metrics (loss, AUC, F1, recall)
├── train.log              # stdout log
├── results.json           # Final test metrics + confusion matrix
```
